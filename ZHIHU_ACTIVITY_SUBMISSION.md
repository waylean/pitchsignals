## 项目亮点

1. 不只输出一个结果，而是展示“为什么倾向这个结果”。
2. 数据层覆盖赛程、赛前市场数据、球队实力、阵容伤停、战术对位、裁判环境、天气、新闻预览和非官方信息。
3. LLM 不直接拍板，而是负责问题拆解、资料结构化和证据审阅。
4. 对缺失数据显式降权，避免假装信息完整。
5. 支持本地部署和非商业开源，适合研究、教学和二次开发。
6. 默认使用公开免费数据源，并可选接入知乎开放平台搜索，增强中文社区语境下的信息检索。

## 它能做什么

打开项目后，先点击“获取赛程”，系统会拉取今明两天可分析的足球比赛。选择一场比赛后，点击预测，就会进入一套完整的赛前研究流程。

当前会参考的信息包括：

- 赛程和比赛时间
- 赛前市场数据
- 球队长期实力
- 阵容与伤停
- 战术对位
- 裁判与环境
- 天气和地点
- 公开新闻、赛前预览、非官方消息
- 中文社区语境下的公开讨论

最后输出胜、平、负的倾向概率、综合置信度和每类参考数据的权重。

## 我最想解决的问题

很多预测项目最大的问题是：只给一个结论，不告诉你结论从哪里来。

PitchSignals 反过来做。它会先把一场比赛拆成几个可解释的部分：

1. 数据层：尽可能收集公开且免费的赛前信息。
2. 预测权重层：根据历史比赛和回测经验，决定不同因子应该占多大权重。
3. 预测层：综合结构化数据、权重和置信度，输出胜平负倾向。
4. 反馈层：比赛结束后可以回填真实结果，继续分析模型哪里判断错了。

LLM 在里面不是最终裁判。它主要负责把难处理的文本资料变成结构化信号，比如把一篇赛前报道里的伤停、战术倾向、阵容变化提取出来。真正的概率计算仍然由可审计的权重模型完成。

## 为什么要接入知乎开放平台

足球比赛的信息不只存在于英文数据源里。很多赛前讨论、球迷观察、中文报道和社区观点，会出现在中文内容平台。

所以我把知乎开放平台搜索作为一个可选增强数据源接入到搜索层。它不会替代默认的公开搜索链路，而是作为中文语境补充：当用户配置了知乎开放平台 Access Secret 后，系统可以额外检索中文内容，并把结果作为候选证据进入结构化流程。

如果没有配置知乎接口，项目仍然可以依靠默认公开数据源运行。

## 这个项目的边界

PitchSignals 是研究工具，不承诺结果一定正确。足球比赛本来就有大量偶然性，红牌、伤病、临场调整、心理压力都会改变比赛。

所以我更看重的是：它能不能帮助用户理解一场比赛，能不能把信息来源、权重和不确定性讲清楚。

这个项目只用于研究、学习和娱乐，不用于商业预测服务或任何高风险决策。

## 如何运行

本地需要：

- Python 3.11+
- Node.js 18+
- 一个兼容 OpenAI API 协议的 LLM API key
- 一个免费的 The Odds API key
- 可选：知乎开放平台 Access Secret

仓库里提供了 `AGENT_DEPLOY.md`。如果你使用 Codex、Cursor、Claude Code 或其他编程 Agent，只需要替换 API key，把整段部署指令发给 Agent，就可以本地启动后端和前端。
